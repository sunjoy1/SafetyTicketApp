# 安全管理作业票系统 - 打包安装指南

## 方式一：用 Android Studio 图形界面打包（推荐，最简单）

### 步骤 1：下载安装 Android Studio

1. 打开浏览器，访问 https://developer.android.com/studio
2. 下载 **Windows (64-bit) 版本**（约 1GB）
3. 双击安装包，一路点击「Next」默认安装即可（安装过程约 10-20 分钟）

> ⚠️ 安装时建议勾选 **Android SDK**、**Android Virtual Device** 和 **Performance (Android Emulator)**

---

### 步骤 2：打开项目

1. 打开 Android Studio
2. 选择 **Open**（不要选 New Project）
3. 找到并选择你的项目文件夹：
   ```
   C:\Users\suman\Documents\kimi\workspace\SafetyTicketApp
   ```
4. 点击 **OK**，等待 Gradle 自动同步（底部有进度条，第一次可能需要 5-15 分钟，取决于网速）

> 如果提示下载 Gradle 或 SDK 组件，点击 **Accept** 和 **Install** 即可。

---

### 步骤 3：生成 APK 文件

1. 点击顶部菜单栏：**Build → Build Bundle(s) / APK(s) → Build APK(s)**
2. 等待编译完成（右下角会显示进度）
3. 编译完成后，右下角弹出提示：**Build Analyzer detected...**，点击旁边的 **locate** 或 **Show in Explorer**
4. 找到 APK 文件：
   ```
   app\build\outputs\apk\debug\app-debug.apk
   ```

---

### 步骤 4：安装到手机

**方法 A：通过 Android Studio 直接安装**
1. 用 USB 连接手机
2. 手机设置 → 关于手机 → 连续点击「版本号」7次 → 开启开发者选项
3. 设置 → 系统和更新 → 开发者选项 → 开启 **USB 调试**
4. 回到 Android Studio，点击顶部工具栏的绿色 **Run** 按钮（▶）
5. 选择你的手机 → 自动编译安装

**方法 B：发送 APK 到手机安装**
1. 把 `app-debug.apk` 文件通过微信文件传输、QQ、数据线等方式传到手机
2. 在手机上打开 APK 文件
3. 系统可能提示「安装未知来源应用」，点击 **允许** 或去设置里开启
4. 安装完成

---

## 方式二：用脚本一键打包（适合批量或给别人打包）

### 步骤 1：先完成上面的步骤 1 和 2（安装 Android Studio + 打开项目）

### 步骤 2：找到你的 SDK 路径

1. 在 Android Studio 中，点击菜单 **File → Project Structure**
2. 在弹出的窗口中，找到 **SDK Location** → **Android SDK location**
3. 复制路径（类似这样）：
   ```
   C:\Users\你的用户名\AppData\Local\Android\Sdk
   ```

### 步骤 3：修改脚本中的 SDK 路径

1. 在项目文件夹中找到 `一键打包.bat`
2. 右键 → **编辑**（用记事本打开）
3. 找到第 20 行：
   ```bat
   set SDK_PATH=C:\Users\%USERNAME%\AppData\Local\Android\Sdk
   ```
4. 如果你的 SDK 路径不是默认的，修改成你的实际路径，例如：
   ```bat
   set SDK_PATH=D:\Android\Sdk
   ```
5. 保存并关闭

### 步骤 4：运行脚本

1. 双击运行 `一键打包.bat`
2. 脚本会自动编译，并将 APK 复制到桌面
3. 桌面会出现：`安全管理作业票系统.apk`
4. 发送这个文件到手机即可安装

## 常见问题

### 问题 1：编译报错 "Could not find com.android.tools.build:gradle:8.2.2"

**解决**：这是 Gradle 依赖下载失败。需要确保网络可以访问 Google 服务器，或者配置国内镜像：

1. 打开 `build.gradle`（项目根目录那个）
2. 把 `repositories` 改成：
   ```gradle
   repositories {
       google()
       mavenCentral()
       maven { url 'https://maven.aliyun.com/repository/google' }
       maven { url 'https://maven.aliyun.com/repository/central' }
   }
   ```
3. 重新点击 **Build → Rebuild Project**

### 问题 2：编译报错 "Minimum supported Gradle version is 8.2"

**解决**：Gradle 版本不匹配。在 Android Studio 中：

1. 点击菜单 **File → Project Structure → Project**
2. 确保 **Gradle Version** 和 **Android Gradle Plugin Version** 匹配
3. 或者点击 **File → Invalidate Caches / Restart** → 选择 **Invalidate and Restart**

### 问题 3：编译报错 "No toolchains found..."

**解决**：缺少 SDK Build Tools。在 Android Studio 中：

1. 点击菜单 **Tools → SDK Manager**
2. 勾选 **SDK Tools** 标签页下的 **Android SDK Build-Tools** 和 **Android SDK Platform-Tools**
3. 点击 **Apply** 安装

### 问题 4：APK 安装后没有图标

**解决**：当前版本为了简化编译去掉了图标。你可以自行添加：

1. 在 Android Studio 中，右键 `res` 文件夹 → **New → Image Asset**
2. 选择一张 PNG 图片作为图标，自动生成各种尺寸的图标文件
3. 重新编译

### 问题 5：运行后点击「现场拍照」闪退

**解决**：缺少相机权限。首次运行会弹窗申请权限，请点击 **允许**。如果禁止了，去手机设置 → 应用管理 → 安全管理作业票系统 → 权限 → 开启相机和存储权限。

## 如果上面的方法都搞不定

请把你的项目文件压缩成 ZIP，发送给有 Android Studio 的同事/朋友，让他们帮你打包一次，之后再把 APK 发回给你。只要打包一次成功，以后你可以直接复制桌面上的 APK 发给其他人安装。
